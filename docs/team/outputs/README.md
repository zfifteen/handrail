# Virtual Team Outputs

This directory is the stable output location for recurring Handrail virtual team reports.

Expected report files:

- `pm.md`
- `architect.md`
- `qa-lead.md`
- `lead-dev.md`

Reports should be replaced with the latest role assessment unless the role has a specific reason to preserve a dated run log. A report is not required when the role's useful output is a code change, test change, documentation change, or GitHub issue update, but the role must still leave a clear final summary in the automation run.

Every report should state the strongest supported finding first, then list the concrete action, touched files or issues, remaining blocker, and verification performed.
